## 🎯 项目概述

成功实现了一个完整的"庭审式 Agent 协作系统"（Courtroom Collaboration），用对抗式辩论替代传统的消息队列，为多 Agent 协作注入"程序正义"。

### 核心理念

> "我们的 Agent 不吵架，只开庭。"

将多 Agent 协作转化为法庭庭审流程，通过对抗式辩论确保决策质量，用程序正义保障协作安全。

### 1. 核心框架 ⚖️

**文件**: `courtroom/schemas.py`, `courtroom/court.py`

### 2. Agent 角色 👥

**文件**: `courtroom/agents/*.py`

#### 规则引擎版本
- ✅ **法官 (Judge)**: 唯一有权修改共享状态，做出最终裁决
- ✅ **检察官 (Prosecutor)**: 提出变更提案，推进任务
- ✅ **辩护律师 (Defender)**: 质疑提案，发现潜在问题
- ✅ **陪审团 (Jury)**: 4 位专家（安全、性能、测试、架构）
- ✅ **书记员 (CourtReporter)**: 生成人类可读的庭审记录

#### LLM 增强版本
- ✅ **ProsecutorLLM**: 使用 Claude API 生成智能论点
- ✅ **DefenderLLM**: 使用 LLM 发现深层问题
- ✅ **JudgeLLM**: 综合分析做出公正判决
- ✅ 自动回退机制：LLM 失败时使用规则引擎
